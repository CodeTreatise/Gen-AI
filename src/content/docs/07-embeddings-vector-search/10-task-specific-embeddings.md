# Task-Specific Embeddings

- Why task type matters
  - Same text, different optimal embedding
  - Query vs document asymmetry
  - Task-optimized vector spaces
  - Significant quality improvements
- Gemini task types (comprehensive)
  - RETRIEVAL_QUERY
    - For search queries/questions
    - Optimized for matching to documents
    - Use for user input in search
  - RETRIEVAL_DOCUMENT
    - For documents being indexed
    - Asymmetric to RETRIEVAL_QUERY
    - Use when building search index
  - SEMANTIC_SIMILARITY
    - For comparing text pairs
    - Symmetric comparisons
    - Use for duplicate detection, paraphrase
  - CLASSIFICATION
    - For text categorization
    - Optimized for class separation
    - Use for training classifiers
  - CLUSTERING
    - For grouping similar items
    - Optimized for cluster separation
    - Use for topic modeling, grouping
  - CODE_RETRIEVAL_QUERY
    - For code search queries
    - Understands natural language to code
    - Use for code search systems
  - QUESTION_ANSWERING
    - For Q&A system queries
    - Optimized for answer retrieval
    - Use for FAQ systems
  - FACT_VERIFICATION
    - For fact-checking queries
    - Evidence retrieval optimization
    - Use for claim verification
- Cohere input_type parameter
  - search_query: for search queries
  - search_document: for searchable content
  - classification: for categorization
  - clustering: for grouping
  - image: for image embeddings
- OpenAI approach
  - No explicit task type parameter
  - Single general-purpose model
  - Works well across tasks
  - Consider prefix strategies
- Best practices
  - ALWAYS match query/document types
  - Don't mix task types in same index
  - Benchmark task types for your use case
  - Document your task type choices
