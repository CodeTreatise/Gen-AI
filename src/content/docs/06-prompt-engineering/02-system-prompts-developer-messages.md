# System Prompts & Developer Messages

- Purpose of system prompts
  - Persistent behavior definition
  - Separated from user input
  - Higher priority than user messages
  - Setting unchangeable rules
- **Developer Role (Responses API)**
  - OpenAI Responses API uses `developer` role instead of `system`
  - `instructions` parameter for high-level guidance
  - Developer messages take precedence over user messages
  - Enables clearer separation of app logic and user input
  - Example: `{ "role": "developer", "content": "You are a helpful assistant" }`
- **System vs Developer Role Comparison**
  - Chat Completions API: `system` role
  - Responses API: `developer` role + `instructions` param
  - Anthropic Claude: `system` parameter (not in messages array)
  - Google Gemini: `systemInstruction` parameter
- Defining AI persona and behavior
  - Character and personality
  - Expertise areas
  - Communication style
  - Behavioral boundaries
- Setting response constraints
  - Length limits
  - Format requirements
  - Forbidden topics
  - Required elements
- Establishing tone and style
  - Formal vs. casual
  - Technical level
  - Humor and personality
  - Consistency maintenance
- Safety guidelines in system prompts
  - Content restrictions
  - Harmful request handling
  - Privacy protection
  - Jailbreak resistance
- System prompt length considerations
  - Token budget allocation
  - Essential vs. optional rules
  - Prompt caching benefits
  - Performance impact
