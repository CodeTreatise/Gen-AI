# Defining Functions

- Function definition structure
  - Name, description, parameters
  - Schema format requirements
  - `type: "function"` wrapper
  - `strict: true` for guaranteed schema adherence
- Provider-specific definition formats
  - OpenAI: tools array with function objects
  - Anthropic: tools array with input_schema
  - Gemini: function_declarations in Tool object
  - Format conversion between providers
- Function naming conventions
  - Verb-noun patterns (get_weather, send_email)
  - No spaces, periods, or dashes
  - Underscores or camelCase
  - Avoiding ambiguity
- Description writing for AI understanding
  - Clear purpose statement
  - When to use guidance
  - Parameter explanation
  - Example scenarios in description
  - Edge cases documentation
- When AI should use each function
  - Trigger conditions
  - Alternative functions
  - Edge case handling
  - System prompt guidance for tool selection
- Function count and token management
  - Functions count against context limit
  - Billed as input tokens
  - Aim for 10-20 functions maximum
  - Performance degrades with too many tools
  - Dynamic tool selection for large tool sets
- Function organization strategies
  - Grouping related functions
  - Category-based organization
  - Context-aware tool filtering
  - Fine-tuning for function calling accuracy
