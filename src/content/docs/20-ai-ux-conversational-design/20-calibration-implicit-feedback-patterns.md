# Calibration & Implicit Feedback Patterns

- Initial calibration UX
  - Quick, minimal data collection
  - Clear benefit explanation
  - Progress and completion confirmation
  - Cancelation without judgment
- Implicit feedback collection
  - Learn from user behavior without asking
  - Secure sensitive information
  - Avoid echo chamber / confirmation bias
  - Balance reinforcement with exploration
- Explicit feedback requests
  - Voluntary, non-intrusive
  - Simple, direct language
  - Immediate action on feedback
  - Persistent changes
- Corrections interface
  - Familiar editing patterns
  - Immediate value display
  - Learn from corrections (when appropriate)
  - Guided vs. freeform corrections
